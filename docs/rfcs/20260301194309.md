# RFC 20260301194309: Fix "found undefined" yomitan version error and improve connectivity

## Introduction
When using external tools like asbplayer together with the Yomitan API, some clients were reporting that the Yomitan version was `undefined` (e.g., `Minimum Yomitan version is 25.12.16.0, found undefined`). This issue prevented integration features from working correctly even when the API was active.

## Objectives
- Ensure that version checks (both for the server and for the Yomitan extension) are accessible via `GET` requests.
- Improve error reporting when the Native Messaging connection to the browser is broken or inactive.
- Prevent manual execution errors that disrupt existing browser-managed API instances.

## Implementation Details

### 1. GET Request Support
The `RequestHandler` in `yomitan_api.py` was refactored to support both `GET` and `POST` methods via a unified `do_request()` method.
- **Paths**: `/serverVersion` and `/` now correctly return the server's version in JSON format when queried with a `GET` request.
- **Body Parsing**: Added logic to bypass body reading for `GET` requests while maintaining it for `POST`.

### 2. Native Messaging Error Handling
When the API is run manually or when the browser pipe is disconnected, `sys.stdout.buffer.write` can fail with an `OSError` (particularly `[Errno 22]` on Windows).
- **503 Service Unavailable**: The server now catches `OSError` and `BrokenPipeError` during `send_message`, returning a `503` status code with a descriptive message explaining that the Native Messaging connection failed.
- **504 Gateway Timeout**: If the server is connected but the Yomitan extension fails to respond, a `504` error is returned instead of a generic `500`.

### 3. TTY Awareness & Instance Management
To prevent users from accidentally starting a manual instance that kills the "official" browser-managed instance (and breaks the pipe), the server now detects if it is running in an interactive TTY.
- If a TTY is detected and another instance is already running (checked via `.crowbar`), the script warns the user and exits without killing the daemon.

### 4. Installer Refactor
The `install_yomitan_api.py` script was wrapped in `if __name__ == "__main__":` to allow its pure functions to be imported into unit tests without triggering interactive side-effects.

## Analytics and Decisions
- **Consistency**: Decided to use standard HTTP status codes (503/504) to represent internal connectivity problems, providing more actionable feedback than a generic `500`.
- **Safety**: Chose to prioritize protecting existing instances over allowing manual overrides in TTY mode, as the most common issue was users accidentally running the script manually and "stealing" the port from the browser instance.
- **Testability**: Set `PROCESS_STARTUP_WAIT = 0` globally in tests to ensure fast execution without artificial delays.

## Verification
- **Automated Tests**: Updated `tests/test_server.py` and `tests/test_install.py` with comprehensive mocks.
- **Manual Smoke Test**: Verified `GET /serverVersion` returns `{"version": 1}`.
- **Error Simulation**: Verified that killing the browser pipe results in a `503` response for subsequent API calls.
